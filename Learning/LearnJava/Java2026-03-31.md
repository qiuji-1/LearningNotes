## 1. 什么是Java中的类

### 📌 回答重点

**类是对象的模板/蓝图**，定义了对象应该有什么属性（字段）和行为（方法）。

**核心概念：**
- 类是抽象的概念，不是具体的东西
- 类定义了"这类对象应该长什么样"
- 类包含：属性（字段）+ 行为（方法）

---

### 💡 面试官最爱问的3个细节

#### 1) 类的基本结构

```java
public class Student {
    // 属性（字段）
    String name;
    int age;
    
    // 行为（方法）
    public void study() {
        System.out.println(name + "正在学习");
    }
}
```

#### 2) 类的三要素

| 要素 | 说明 | 示例 |
|------|------|------|
| 字段 | 对象的数据/属性 | `String name` |
| 方法 | 对象的行为/功能 | `void study()` |
| 构造器 | 创建对象时初始化 | `public Student()` |

#### 3) 类与对象的关系

- 类 = 模具/图纸
- 对象 = 根据模具生产的产品
- 一个类可以创建无数个对象

#### 4) 为什么一个类可以创建无数个对象？

**示例代码：**
```java
public class Student {
    // 属性（字段）
    String name;
    int age;
    
    // 行为（方法）
    public void study() {
        System.out.println(name + "正在学习");
    }
}
```

**在这里：**
- `Student` 是**类**（Class）- 模板
- `Student` 不是对象

**如何区分类和对象：**

| 特征 | 类 | 对象 |
|------|------|------|
| 定义 | 使用 `class` 关键字定义 | 使用 `new` 关键字创建 |
| 本质 | 模板、蓝图 | 实例、具体实体 |
| 抽象性 | 抽象的概念 | 具体的存在 |
| 可创建性 | 不能被创建 | 可以被类创建 |
| 内存分配 | 只加载一次到方法区 | 每次创建都分配堆内存 |

**创建多个对象的示例：**

```java
// 创建多个对象
Student stu1 = new Student();  // 第一个对象
Student stu2 = new Student();  // 第二个对象
Student stu3 = new Student();  // 第三个对象

// 每个对象都是独立的，有自己的属性
stu1.name = "张三";
stu2.name = "李四";
stu3.name = "王五";

stu1.age = 18;
stu2.age = 19;
stu3.age = 20;

// 每个对象调用方法，输出不同结果
stu1.study();  // 张三正在学习
stu2.study();  // 李四正在学习
stu3.study();  // 王五正在学习
```

**为什么可以创建无数个对象？**

1. **类是模板，不是实体**
   - 类定义了对象的"结构"和"行为"
   - 就像房屋图纸可以无限次使用建造多栋房子

2. **每个对象独立分配内存**
   ```
   方法区：
   ┌──────────────────────────┐
   │   Student类（模板）        │
   │   - name字段              │
   │   - age字段               │
   │   - study()方法           │
   └──────────────────────────┘
           │ 复用
           ▼
   堆内存：
   ┌─────────┐  ┌─────────┐  ┌─────────┐
   │ stu1对象│  │ stu2对象│  │ stu3对象│
   │ 张三    │  │ 李四    │  │ 王五    │
   │ 18      │  │ 19      │  │ 20      │
   └─────────┘  └─────────┘  └─────────┘
   ```

3. **类比理解**
   - 类 = 月饼模具
   - 对象 = 用模具压出来的每个月饼
   - 模具只有一个，但可以压出无数个月饼
   - 每个月饼独立存在，有自己的味道和形状

---

### 📝 总结

| 概念 | 说明 |
|------|------|
| 类 | 模板，定义对象的结构和行为 |
| 字段 | 对象的属性数据 |
| 方法 | 对象的行为功能 |
| 构造器 | 创建并初始化对象 |

## 2. 什么是实例化

### 📌 回答重点

**实例化就是根据类创建具体的对象**。类只是模板，要使用它需要创建实例（对象）。

**核心流程：**
1. 声明引用变量
2. 使用 `new` 关键字创建对象
3. 调用构造器初始化对象
4. 将对象地址赋给引用变量

---

### 💡 面试官最爱问的3个细节

#### 1) 实例化的完整过程

```java
Student student1 = new Student();  // 这就是实例化
//   ↑            ↑       ↑
// 引用变量    创建对象   调用构造器
```

#### 2) 对象与引用的区别

```java
Student s1 = new Student();  // s1是引用，new Student()才是对象
Student s2 = s1;             // s2和s1指向同一个对象
```

- **引用**：存储对象的地址（像遥控器）
- **对象**：真正存储数据的实体（像电视机）
- 一个对象可以被多个引用指向

#### 3) 静态初始化、字段初始化、构造器执行的区别

**完整的初始化顺序：**

```
类加载时（只执行一次）
    ↓
1. 静态变量初始化
2. 静态代码块执行
    ↓
创建对象时（每次创建都执行）
    ↓
3. 实例变量初始化
4. 实例代码块执行
5. 构造器执行
```

**代码示例：**

```java
public class Student {
    // 静态变量（类加载时初始化，只执行一次）
    static int count = 0;
    
    // 实例变量（每次创建对象时初始化）
    String name = "默认值";
    
    // 静态代码块（类加载时执行，只执行一次）
    static {
        System.out.println("1. 静态代码块执行");
        count = 100;
        System.out.println("   count = " + count);
    }
    
    // 实例代码块（每次创建对象时执行）
    {
        System.out.println("3. 实例代码块执行");
        name = "代码块赋值";
    }
    
    // 构造器（每次创建对象时执行，最后执行）
    public Student() {
        System.out.println("4. 构造器执行");
        name = "构造器赋值";
    }
    
    public static void main(String[] args) {
        System.out.println("===== 创建第一个对象 =====");
        Student s1 = new Student();
        System.out.println("s1.name = " + s1.name);
        
        System.out.println("\n===== 创建第二个对象 =====");
        Student s2 = new Student();
        System.out.println("s2.name = " + s2.name);
    }
}
```

**输出结果：**
```
1. 静态代码块执行
   count = 100
===== 创建第一个对象 =====
3. 实例代码块执行
4. 构造器执行
s1.name = 构造器赋值

===== 创建第二个对象 =====
3. 实例代码块执行
4. 构造器执行
s2.name = 构造器赋值
```

**关键点：**
- 静态代码块只在类加载时执行一次
- 静态变量也只在类加载时初始化一次
- 实例代码块和构造器每次创建对象都会执行
- 实例变量每次创建对象都会初始化

---

### 三者的详细区别

| 特性 | 静态初始化 | 字段初始化 | 构造器执行 |
|------|-----------|-----------|-----------|
| **包括** | 静态变量、静态代码块 | 实例变量、实例代码块 | 构造器方法 |
| **执行时机** | 类加载时（只一次） | 每次创建对象时 | 每次创建对象时（最后） |
| **执行顺序** | 最先执行 | 在构造器之前 | 最后执行 |
| **存储位置** | 方法区 | 堆内存 | 堆内存 |
| **归属** | 属于类 | 属于对象 | 属于对象 |
| **可访问** | 静态成员 | 实例成员 + 静态成员 | 实例成员 + 静态成员 |
| **this关键字** | ❌ 不能用 | ✅ 可以用 | ✅ 可以用 |

### 更多示例

**示例1：多个静态代码块和实例代码块的执行顺序**

```java
public class Test {
    static int a;
    static int b;
    int x;
    int y;
    
    // 静态代码块1

        System.out.println("静态代码块2");
        b = 2;
    }
    
    // 实例代码块1
    {
        System.out.println("实例代码块1");
        x = 10;
    }
    
    // 实例代码块2
    {
        System.out.println("实例代码块2");
        y = 20;
    }
    
    public Test() {
        System.out.println("构造器");
    }
    
    public static void main(String[] args) {
        System.out.println("===== 创建第一个对象 =====");
        Test t1 = new Test();
        System.out.println("===== 创建第二个对象 =====");
        Test t2 = new Test();
    }
}
```

**输出：**
```

静态代码块2
===== 创建第一个对象 =====
实例代码块1

构造器

实例代码块1
实例代码块2
构造器
```

**执行顺序：**
1. 静态代码块按定义顺序执行（只执行一次）
2. 实例代码块按定义顺序执行（每次创建对象都执行）
3. 最后执行构造器

## 3. static关键字的作用

### 📌 回答重点

**static表示"属于类本身，不属于对象实例"**。

**核心特点：**
- 被所有实例共享，只有一份
- 可以直接通过类名访问
- 随类加载而初始化，不需要创建对象

---

### 💡 面试官最爱问的3个细节

#### 1) 静态变量 vs 实例变量

```java
public class Student {
    String name;           // 实例变量：每个对象独有
    static int count = 0;  // 静态变量：所有对象共享
}
```

| 类型 | 归属 | 数量 | 访问方式 |
|------|------|------|----------|
| 实例变量 | 对象 | 每个对象一份 | `对象.变量` |
| 静态变量 | 类 | 只有一份 | `类名.变量` |

#### 2) 静态方法的特点

```java
public class Student {
    String name;
    static int count = 0;
    
    // 静态方法：可直接用类名调用
    public static void showCount() {
        System.out.println("学生总数：" + count);
        // System.out.println(name);  // 错误！不能访问实例变量
    }
    
    // 实例方法：必须用对象调用
    public void study() {
        System.out.println(name + "正在学习");
        showCount();  // 可以访问静态方法
    }
}
```

**静态方法限制：**
- ✅ 可以访问静态变量和静态方法
- ❌ 不能访问实例变量和实例方法
- ❌ 不能使用 `this` 关键字

#### 3) 实际应用示例

```java
public class Student {
    String name;
    static int count = 0;
    
    public Student() {
        count++;  // 统计创建了多少个学生
    }
    
    public static void showCount() {
        System.out.println("学生总数：" + count);
    }
}

// 使用
Student s1 = new Student();
Student s2 = new Student();
Student.showCount();  // 输出：学生总数：2
```

---

### 📝 总结

| 关键点 | 说明 |
|--------|------|
| 静态变量 | 所有对象共享一份，用 `类名.变量` 访问 |
| 静态方法 | 直接用 `类名.方法()` 调用，不能访问实例成员 |
| 静态代码块 | `static { }` 类加载时执行一次 |
| 应用场景 | 工具方法、计数器、常量 |

---

## 4. 接口和抽象类有什么区别

### 📌 回答重点

接口和抽象类在设计动机上有所不同：

**接口（自上而下）：**
- 我们知晓某一行为，于是基于这些行为约束定义了接口
- 一些类需要有这些行为，因此实现对应的接口
- 先约定接口，再实现

**抽象类（自下而上）：**
- 我们写了很多类，发现它们之间有共性，有很多代码可以复用
- 因此将公共逻辑封装成一个抽象类，减少代码冗余
- 先有一些类，才抽象了共同父类

> 💡 实战中很多时候都是因为重构才有的抽象

---

### 💡 面试官最爱问的3个细节

#### 1) 方法实现

```java
// 接口：Java 8之前只能有抽象方法
public interface Flyable {
    void fly();  // 默认是public abstract
    
    // Java 8之后可以有default方法和静态方法
    default void defaultMethod() {
        System.out.println("接口的默认方法");
    }
    
    static void staticMethod() {
        System.out.println("接口的静态方法");
    }
}

// 抽象类：可以有抽象方法和具体方法
public abstract class Animal {
    public abstract void eat();  // 抽象方法
    
    public void sleep() {        // 具体方法
        System.out.println("动物在睡觉");
    }
}
```

#### 2) 构造函数和成员变量

| 特性 | 接口 | 抽象类 |
|------|------|--------|
| 构造函数 | ❌ 不能有 | ✅ 可以有 |
| 成员变量 | `public static final`（常量） | 可以是各种访问修饰符 |
| 初始化 | 只能有常量 | 可以有实例变量 |

```java
// 接口：成员变量默认是public static final
public interface Constants {
    int MAX_VALUE = 100;  // 等价于 public static final int MAX_VALUE = 100
}

// 抽象类：成员变量可以有多种修饰符
public abstract class Animal {
    private String name;           // 私有变量
    protected int age;             // 保护变量
    public static int count = 0;   // 静态变量
    
    public Animal(String name) {   // 构造函数
        this.name = name;
    }
}
```

#### 3) 多继承

```java
// 抽象类：只能单继承
public abstract class Animal { }
public abstract class Mammal { }
// public class Dog extends Animal, Mammal { }  // ❌ 错误！不能多继承

// 接口：可以多实现
public interface Flyable { }
public interface Swimmable { }
public class Duck implements Flyable, Swimmable { }  // ✅ 正确！可以多实现
```

---

### 📝 总结

| 区别 | 接口 | 抽象类 |
|------|------|--------|
| 设计动机 | 自上而下（先定义行为） | 自下而上（后提取共性） |
| 方法 | 默认public abstract，Java 8后可有default/static | 可以有抽象和具体方法 |
| 变量 | 只能是public static final常量 | 可以是任意访问修饰符 |
| 构造器 | ❌ 不能有 | ✅ 可以有 |
| 继承 | 可以多实现 | 只能单继承 |

---

### 🔍 拓展知识：抽象类 vs 普通类 & 三种方法详解

#### 一、抽象类 vs 普通类

```java
// 普通类：可以直接实例化
public class Animal {
    public void eat() {
        System.out.println("动物在吃");
    }
}
Animal a = new Animal();  // ✅ 可以

// 抽象类：不能直接实例化
public abstract class Animal {
    public abstract void eat();  // 抽象方法
}
Animal a = new Animal();  // ❌ 错误！不能实例化抽象类
```

**关键区别：**

| 特性 | 抽象类 | 普通类 |
|------|--------|--------|
| 实例化 | ❌ 不能直接new | ✅ 可以直接new |
| 抽象方法 | ✅ 可以有 | ❌ 不能有 |
| 被继承 | 子类必须实现所有抽象方法（除非子类也是抽象类） | 子类可以选择性重写方法 |
| 设计目的 | 作为基类，定义通用行为模板 | 具体的实现类 |

---

#### 二、抽象方法、default方法、static方法

##### 1) 抽象方法（abstract）

**定义：** 只有声明，没有实现，必须由子类重写。

```java
// 抽象类中的抽象方法
public abstract class Animal {
    public abstract void eat();  // 没有方法体
}

// 接口中的抽象方法
public interface Flyable {
    void fly();  // 默认就是public abstract
}
```

**特点：**
- ❌ 不能有方法体
- ❌ 不能是private（子类需要重写）
- ❌ 不能是static（静态方法不能被重写）
- ✅ 必须被子类实现（除非子类也是抽象类）

---

##### 2) default方法（Java 8+）

**定义：** 接口中的默认实现方法，子类可以继承或重写。

```java
public interface Flyable {
    // default方法：有默认实现
    default void fly() {
        System.out.println("默认飞行方式");
    }
}

// 实现类可以选择不重写
public class Bird implements Flyable {
    // 不写fly()也能用，继承默认实现
}

// 实现类也可以重写
public class Eagle implements Flyable {
    @Override
    public void fly() {
        System.out.println("老鹰飞得很高");
    }
}
```

**特点：**
- ✅ 有方法体
- ✅ 实现类可以不重写，使用默认实现
- ✅ 实现类可以重写
- ✅ 解决了接口演进问题（给接口添加新方法不影响已有实现类）

---

##### 3) static方法（Java 8+）

**定义：** 接口或类中的静态方法，属于类型本身。

```java
public interface Flyable {
    // 静态方法
    static void showInfo() {
        System.out.println("这是飞行接口");
    }
}

// 调用方式
Flyable.showInfo();  // 用接口名调用，不能用实现类调用

public class Utils {
    public static void log(String msg) {
        System.out.println("[LOG] " + msg);
    }
}
Utils.log("测试");  // 普通类的静态方法
```

**特点：**
- ✅ 有方法体
- ❌ 不能被重写（static方法不支持多态）
- ❌ 实现类不能继承接口的static方法
- ✅ 只能用`接口名.方法名()`或`类名.方法名()`调用

---

##### 4) 三者对比表

| 特性 | 抽象方法 | default方法 | static方法 |
|------|----------|-------------|------------|
| 有无方法体 | ❌ 无 | ✅ 有 | ✅ 有 |
| 能否被重写 | ✅ 必须重写 | ✅ 可选重写 | ❌ 不能 |
| 调用方式 | 通过对象调用 | 通过对象调用 | 通过类名/接口名调用 |
| 出现位置 | 抽象类、接口 | 接口（Java 8+） | 类、接口（Java 8+） |
| 多态支持 | ✅ 支持 | ✅ 支持 | ❌ 不支持 |

---

##### 5) 实际应用场景

```java
// 接口定义
public interface Database {
    // 抽象方法：子类必须实现
    void connect();
    
    // default方法：提供默认实现，子类可选重写
    default void log(String msg) {
        System.out.println("[DB] " + msg);
    }
    
    // static方法：工具方法，不属于某个实例
    static String getVersion() {
        return "1.0.0";
    }
}

// 实现类
public class MySQL implements Database {
    @Override
    public void connect() {
        System.out.println("连接MySQL");
    }
    
    // log()可以不重写，使用默认实现
}

// 使用
MySQL db = new MySQL();
db.connect();              // 调用抽象方法
db.log("查询数据");         // 调用default方法
Database.getVersion();      // 调用static方法
```

## 7. 关于Java的反射机制，如何应用反射

### 📌 回答重点

反射机制就是让程序在运行时能够"照镜子"看自己，动态获取类的结构信息，包括方法、字段、构造函数，还能直接操作它们。

**核心特点：**
- 编译期不需要知道具体类型
- 运行时再决定要用哪个类、调哪个方法
- 打破了封装性，可以访问private成员

**反射的核心流程：**
1. 获取Class对象：通过类名字符串、.class语法或对象实例获取
2. 获取成员信息：从Class对象中获取Field、Method、Constructor
3. 操作目标对象：创建实例、读写字段、调用方法

---

### 💡 面试官最爱问的3个细节

#### 1) 三种获取Class对象的方式

```java
// 方式1：通过类名.class（最安全，编译时检查）
Class<Student> clazz1 = Student.class;

// 方式2：通过对象实例.getClass()
Student student = new Student();
Class<?> clazz2 = student.getClass();

// 方式3：通过Class.forName()（最灵活，运行时动态加载）
Class<?> clazz3 = Class.forName("com.example.Student");

// 三种方式获取的是同一个Class对象
System.out.println(clazz1 == clazz2);  // true
System.out.println(clazz2 == clazz3);  // true
```

**对比：**

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 类名.class | 编译时检查，安全 | 需要知道具体类名 | 已知类名时 |
| 对象.getClass() | 不需要import | 需要先有对象实例 | 有对象时 |
| Class.forName() | 最灵活，可动态加载 | 需要处理异常 | 配置文件、框架 |

---

#### 2) 获得Class对象后可以做的事情

```java
// 1. 获取类的基本信息
Class<?> clazz = Student.class;

String className = clazz.getName();          // 全限定名：com.example.Student
String simpleName = clazz.getSimpleName();   // 简单名称：Student

// 2. 获取构造函数并创建实例
Constructor<?> constructor = clazz.getConstructor(String.class, int.class);
Object instance = constructor.newInstance("张三", 20);

// 3. 获取字段并操作
Field nameField = clazz.getDeclaredField("name");
nameField.setAccessible(true);  // 突破private限制
nameField.set(instance, "李四");
Object value = nameField.get(instance);

// 4. 获取方法并调用
Method studyMethod = clazz.getMethod("study", String.class);
studyMethod.invoke(instance, "数学");  // 调用实例的study方法
```

---

#### 3) 实际应用场景

```java
// 场景1：框架的依赖注入（Spring）
String className = "com.example.UserService";  // 从配置文件读取
Class<?> clazz = Class.forName(className);
Object service = clazz.newInstance();  // 动态创建实例

// 场景2：动态代理
interface UserService {
    void save();
}
UserService proxy = (UserService) Proxy.newProxyInstance(
    clazz.getClassLoader(),
    new Class[]{UserService.class},
    (obj, method, args) -> {
        System.out.println("执行前...");
        Object result = method.invoke(target, args);
        System.out.println("执行后...");
        return result;
    }
);

// 场景3：JSON序列化/反序列化（Gson、Jackson）
// 反射获取所有字段，遍历赋值
for (Field field : clazz.getDeclaredFields()) {
    field.setAccessible(true);
    field.set(obj, value);
}
```

---

### 📝 总结

| 类/方法 | 作用 |
|---------|------|
| `Class.forName()` | 动态加载类 |
| `clazz.getConstructor()` | 获取构造函数 |
| `clazz.getMethod()` | 获取方法 |
| `clazz.getField()` | 获取字段 |
| `constructor.newInstance()` | 创建实例 |
| `method.invoke()` | 调用方法 |
| `field.set()/get()` | 读写字段 |
| `setAccessible(true)` | 突破private限制 |

---

### 🔍 拓展知识：反射的优缺点

**优点：**
- ✅ 灵活性高，可以在运行时动态操作
- ✅ 降低代码耦合度
- ✅ 框架必备（Spring、MyBatis、Hibernate）

**缺点：**
- ❌ 性能较低（比直接调用慢很多）
- ❌ 破坏封装性，不安全
- ❌ 代码可读性差
- ❌ 编译期无法检查错误

**性能优化：**
```java
// 使用缓存
private static final Map<String, Method> METHOD_CACHE = new HashMap<>();

public Method getMethod(String name) {
    return METHOD_CACHE.computeIfAbsent(name, 
        key -> clazz.getMethod(key));
}
```

---

### ⚠️ 注意事项

1. **安全性问题**
```java
Field field = clazz.getDeclaredField("password");
field.setAccessible(true);  // 可以访问private字段
// 这会破坏封装，慎用！
```

2. **性能问题**
```java
// 避免频繁反射，可以用缓存
private static final Method METHOD;

static {
    try {
        METHOD = MyClass.class.getMethod("myMethod");
    } catch (NoSuchMethodException e) {
        throw new RuntimeException(e);
    }
}
```

3. **异常处理**
```java
try {
    Class<?> clazz = Class.forName("com.example.Student");
    Object instance = clazz.newInstance();
} catch (ClassNotFoundException e) {
    // 类不存在
} catch (InstantiationException | IllegalAccessException e) {
    // 实例化失败
}
```

## 8. 如果一个线程在Java中被两次调用start()方法，会发生什么

### 📌 回答重点

直接抛 `IllegalThreadStateException`。

Java规定一个Thread对象只能调用一次 `start()`，第二次调用时线程状态已经不是 `NEW` 了，JVM会拒绝启动并抛异常。

**线程状态是单向流转的，一旦离开NEW状态就回不去了。** 这跟操作系统线程的设计有关：一个线程执行完毕后，它的栈空间、寄存器上下文都被回收了，没法复活。

---

### 💡 面试官最爱问的3个细节

#### 1) start()的源码分析

```java
// JDK Thread类的start()源码
public synchronized void start() {
    // 检查线程状态
    if (threadStatus != 0)
        throw new IllegalThreadStateException();

    // 加入线程组
    group.add(this);

    boolean started = false;
    try {
        start0();  // 调用native方法启动线程
        started = true;
    } finally {
        try {
            if (!started) {
                group.threadStartFailed(this);
            }
        } catch (Throwable ignore) {
            // 忽略
        }
    }
}

// native方法
private native void start0();
```

**关键点：**
- `threadStatus != 0` 检查线程状态
- `NEW` 状态下 `threadStatus = 0`
- 一旦启动，`threadStatus` 就变成非0值
- 再次调用时直接抛异常

#### 3) 如何复用线程

既然不能重复启动，那如何复用？

```java
}
```

#### 六种状态详解

```java
public enum State {
    NEW,              // 新建：线程刚创建，还没调用start()
    RUNNABLE,         // 可运行：正在运行或等待CPU时间片
    BLOCKED,          // 阻塞：等待获取锁（synchronized）
    WAITING,          // 等待：无限期等待（wait、join、park）
    TIMED_WAITING,    // 计时等待：有限期等待（sleep、wait、join）
    TERMINATED;       // 终止：线程执行完毕
}
```

#### 各状态示例

```java

// RUNNABLE -> BLOCKED

    // 线程1持有锁，线程2等待锁 -> BLOCKED
}

// RUNNABLE -> WAITING
Object.wait();      // 等待notify()
Thread.join();      // 等待线程结束
LockSupport.park(); // 等待unpark()

// RUNNABLE -> TIMED_WAITING
Thread.sleep(1000);     // 睡眠1秒
Object.wait(1000);      // 等待1秒
Thread.join(1000);      // 等待线程1秒
LockSupport.parkNanos(1000000); // 等待指定纳秒

// RUNNABLE -> TERMINATED
// 线程执行完run()方法，正常结束或异常退出
```

## 9. Java的Optional类是什么，它有什么用

### 📌 回答重点

Optional是Java 8引入的一个容器类，专门用来包装可能为空的值。

**核心作用：**
- 不是消灭空指针，而是让代码更清晰地表达"这个值可能不存在"的语义
- 提供一套链式API简化空值处理

**设计理念：**

Java语言架构师Brian Goetz说得很明白：**Optional是给方法返回值用的，用来表示"没有结果"，而不是返回null。**

---

### 💡 面试官最爱问的对比

#### 对比：传统方式 vs Optional方式

**场景：查找用户，获取用户名称长度**

```java
// ❌ 传统方式：返回null，调用方容易忘记判空
public User findUser(String id) {
    return userDao.findById(id);  // 可能返回null
}

// 调用方（容易忘记判空）
User user = findUser("123");
// 如果user为null，下一行抛NullPointerException
System.out.println(user.getName().length());
```

```java
// ✅ Optional方式：明确表示"可能没有"
public Optional<User> findUser(String id) {
    User user = userDao.findById(id);
    return Optional.ofNullable(user);  // 包装成Optional
}

// 调用方（必须处理空值情况）
Optional<User> userOpt = findUser("123");
String name = userOpt
    .map(User::getName)          // 有值则映射
    .orElse("默认名称");           // 没值则返回默认值
System.out.println(name);
```

#### 2) Optional的常用API

```java

   .filter(val -> val.length() > 3)          // 过滤
   .orElse("默认值");

// 链式调用

   .map(String::toUpperCase)
   .orElse("UNKNOWN");
```

### 🔍 拓展知识：Optional的最佳实践

#### ✅ 推荐用法

```java
// 1. 作为方法返回值
public Optional<User> findById(String id) {
    return Optional.ofNullable(userDao.findById(id));
}

// 2. 链式调用
String name = findById("123")
    .map(User::getName)
    .orElse("默认名称");

// 3. 组合多个Optional
Optional<User> user1 = findById("1");
Optional<User> user2 = findById("2");

String result = user1.or(() -> user2)  // Java 9+
    .map(User::getName)
    .orElse("都没找到");
```

#### ❌ 不推荐用法

```java
// 1. 不要用于字段
public class User {
    private Optional<String> nickname;  // ❌ 不推荐
    private String nickname;            // ✅ 推荐
}

// 2. 不要用于方法参数
public void process(Optional<String> value);  // ❌ 不推荐
public void process(String value);            // ✅ 推荐

// 3. 不要直接调用get()
Optional<User> opt = findById("123");
User user = opt.get();  // ❌ 可能抛异常

// 4. 不要用Optional包装集合
Optional<List<User>> users;  // ❌ 不推荐
List<User> users;            // ✅ 推荐（空集合表示没有数据）
```

### ⚠️ 注意事项

1. **性能问题**：Optional创建对象有开销，不要滥用

```java
// ❌ 每次调用都创建Optional对象
public Optional<String> getName() {
    return Optional.ofNullable(this.name);
}

// ✅ 如果确定不为null，直接返回
public String getName() {
    return this.name;
}
```

2. **序列化问题**：Optional没有实现Serializable

```java
// ❌ 不能直接序列化
public class User implements Serializable {
    private Optional<String> nickname;  // 无法序列化
}
```

3. **使用场景**：主要用于方法返回值

```java
// ✅ 正确：返回可能为空的查询结果
public Optional<User> findById(String id);

// ❌ 错误：用于字段或参数
private Optional<String> name;
public void setName(Optional<String> name);
```

## 10. Java中的I/O流是什么

### 📌 回答重点

I/O流是Java用于处理数据输入输出的机制，是程序与外部世界（文件、网络、控制台）通信的桥梁。

**核心概念：**
- **流（Stream）**：数据的有序序列，像水管一样单向流动
- **输入流（Input）**：从外部读取数据到程序
- **输出流（Output）**：从程序写出数据到外部

**流的分类：**

| 分类维度 | 类型 | 说明 |
|---------|------|------|
| 数据单位 | 字节流 | 处理二进制数据（图片、视频、压缩包） |
|  | 字符流 | 处理文本数据（UTF-8、GBK编码的文本） |
| 流向 | 输入流 | 读取数据 |
|  | 输出流 | 写入数据 |
| 功能 | 节点流 | 直接操作数据源（文件、网络） |
|  | 处理流 | 包装其他流，增强功能（缓冲、转换） |

---

### 💡 面试官最爱问的3个细节

#### 1) 经典的文件读写实例

**使用BufferedReader配合try-with-resources自动关闭资源：**

```java
// 读取文件内容
try (BufferedReader reader = new BufferedReader(new FileReader("config.txt"))) {
    String line;
    while ((line = reader.readLine()) != null) {
        System.out.println(line);
    }
}  // 自动关闭资源，无需finally

// 写入文件内容
try (BufferedWriter writer = new BufferedWriter(new FileWriter("output.txt"))) {
    writer.write("Hello World");
    writer.newLine();
    writer.write("Java I/O");
}  // 自动关闭资源
```

**字节流示例：**

```java
// 复制文件（字节流）
try (FileInputStream fis = new FileInputStream("source.jpg");
     FileOutputStream fos = new FileOutputStream("target.jpg")) {
    
    byte[] buffer = new byte[1024];
    int len;
    while ((len = fis.read(buffer)) != -1) {
        fos.write(buffer, 0, len);
    }
}
```

---

#### 2) 底层原理

**字节流：**
- 直接跟操作系统的文件描述符打交道
- 每次read/write都可能触发系统调用
- 适合处理二进制文件（图片、视频、压缩包）

**字符流：**
- 在字节流基础上加了一层编码转换
- 内部用 `StreamEncoder`/`StreamDecoder` 把字节和字符互相转换
- 默认用平台编码，也可以手动指定UTF-8等字符集
- 适合处理文本文件

```
字符流的工作原理：

程序（字符） 
    ↓
StreamEncoder（编码：字符→字节）
    ↓
字节流（字节）
    ↓
操作系统/文件
```

---

#### 3) 常用I/O流类

**字节流：**

| 类 | 说明 |
|---|------|
| `FileInputStream` / `FileOutputStream` | 文件字节流 |
| `BufferedInputStream` / `BufferedOutputStream` | 缓冲字节流 |
| `DataInputStream` / `DataOutputStream` | 数据流（读写基本类型） |
| `ObjectInputStream` / `ObjectOutputStream` | 对象流（序列化） |

**字符流：**

| 类 | 说明 |
|---|------|
| `FileReader` / `FileWriter` | 文件字符流 |
| `BufferedReader` / `BufferedWriter` | 缓冲字符流 |
| `InputStreamReader` / `OutputStreamWriter` | 转换流（字节↔字符） |
| `PrintWriter` | 打印流（格式化输出） |

#### 1) I/O流的设计思想

**装饰器模式（Decorator Pattern）：**


- 灵活组合，按需添加功能
- 避免类爆炸

#### 2) 缓冲流提速的原理

**为什么缓冲流更快？**

```java
// 无缓冲：每次read()都调用系统调用

int b = fis.read();  // 系统调用

// 有缓冲：先读到内存，再从内存取
BufferedInputStream bis = new BufferedInputStream(fis);
int b = bis.read();  // 从内存缓冲区取，减少系统调用
```

**对比：**

| 方式 | 读取1MB文件 | 系统调用次数 |
|------|------------|--------------|
| 无缓冲 | ~1000次 | ~1000次 |
| 有缓冲（8KB缓冲区） | ~128次 | ~128次 |

**默认缓冲区大小：**
- `BufferedInputStream`：8192字节（8KB）
- `BufferedReader`：8192字符

**手动指定缓冲区大小：**

```java
// 16KB缓冲区
BufferedReader br = new BufferedReader(new FileReader("file.txt"), 16384);
```

#### 4) Java NIO的出现

**传统I/O（BIO）的问题：**
- 阻塞式：一个线程只能处理一个连接
- 线程开销大：1000个连接需要1000个线程
- 效率低：大量线程处于等待状态

**NIO（Non-blocking I/O，Java 1.4）：**

```java
// 传统BIO：阻塞等待
ServerSocket server = new ServerSocket(8080);
Socket socket = server.accept();  // 阻塞等待连接
InputStream is = socket.getInputStream();  // 阻塞等待数据

// NIO：非阻塞
Selector selector = Selector.open();
ServerSocketChannel serverChannel = ServerSocketChannel.open();
serverChannel.configureBlocking(false);  // 设置非阻塞
serverChannel.register(selector, SelectionKey.OP_ACCEPT);

// 一个线程管理多个连接
while (true) {
    selector.select();  // 检查就绪的通道
    Set<SelectionKey> keys = selector.selectedKeys();
    for (SelectionKey key : keys) {
        if (key.isAcceptable()) {
            // 处理连接
        } else if (key.isReadable()) {
            // 处理数据
        }
    }
}
```

**NIO三大组件：**

| 组件 | 说明 |
|------|------|
| Channel（通道） | 双向通道，可读可写 |
| Buffer（缓冲区） | 数据容器，读写都经过缓冲区 |
| Selector（选择器） | 多路复用器，一个线程管理多个通道 |

**对比：**

| 特性 | BIO | NIO |
|------|-----|-----|
| 阻塞 | 阻塞 | 非阻塞 |
| 线程模型 | 一连接一线程 | 一线程多连接 |
| 适用场景 | 连接数少 | 连接数多（高并发） |

## 12. 什么是Java中的自动拆箱和装箱

### 📌 回答重点

**自动装箱（Autoboxing）**：Java编译器自动把基本类型转成对应的包装类型，比如 `int` 变成 `Integer`。

**自动拆箱（Unboxing）**：反过来，把包装类型自动转成基本类型。

这俩机制是Java 5引入的，目的很简单：**让代码写起来更顺手**。以前往集合里塞个数字还得手动 `new Integer(10)`，现在直接塞 `10` 就行了。

---

### 💡 面试官最爱问的3个细节

#### 1) 装箱和拆箱的转换过程

```java
// 装箱过程
int i = 10;
Integer obj = i;  // 自动装箱
// 编译器实际调用：Integer.valueOf(i)

// 拆箱过程
Integer obj = Integer.valueOf(10);
int i = obj;      // 自动拆箱
// 编译器实际调用：obj.intValue()

// 集合中使用
List<Integer> list = new ArrayList<>();
list.add(10);     // 自动装箱：Integer.valueOf(10)
int num = list.get(0);  // 自动拆箱：Integer.intValue()
```

**转换流程：**

```
基本类型 int
    ↓ Integer.valueOf()
包装类型 Integer
    ↓ intValue()
基本类型 int
```

---

#### 2) 底层实现

**装箱：`Integer.valueOf(int)` 源码分析**

```java
public static Integer valueOf(int i) {
    // 缓存范围：-128 ~ 127
    if (i >= IntegerCache.low && i <= IntegerCache.high)
        return IntegerCache.cache[i + (-IntegerCache.low)];
    return new Integer(i);  // 超出缓存范围，new新对象
}

// Integer内部缓存类
private static class IntegerCache {
    static final int low = -128;
    static final int high = 127;
    static final Integer cache[];  // 缓存数组
    
    static {
        // 初始化缓存：-128到127的Integer对象
        cache = new Integer[(high - low) + 1];
        int j = low;
        for(int k = 0; k < cache.length; k++)
            cache[k] = new Integer(j++);
    }
}
```

**关键点：**
- Integer缓存了 `-128 ~ 127` 的对象
- 在这个范围内，`valueOf()` 返回缓存对象
- 超出范围，`new Integer()` 创建新对象

**拆箱：`Integer.intValue()` 源码**

```java
public int intValue() {
    return value;  // 直接返回内部存储的int值
}
```

---

#### 3) 缓存机制的影响

```java
Integer a = 100;
Integer b = 100;
System.out.println(a == b);  // true（缓存对象，同一个引用）

Integer c = 200;
Integer d = 200;
System.out.println(c == d);  // false（超出缓存范围，不同对象）

// 拆箱后比较
System.out.println(c.intValue() == d.intValue());  // true
System.out.println(c.equals(d));  // true（推荐）
```

**所有包装类的缓存范围：**

| 包装类 | 缓存范围 |
|--------|----------|
| Byte | -128 ~ 127（全部） |
| Short | -128 ~ 127 |
| Integer | -128 ~ 127 |
| Long | -128 ~ 127 |
| Character | 0 ~ 127 |
| Boolean | TRUE, FALSE（两个常量） |
| Float | 无缓存 |
| Double | 无缓存 |

---

### 📝 总结

| 概念 | 过程 | 实际调用 |
|------|------|----------|
| 自动装箱 | 基本类型 → 包装类型 | `Integer.valueOf()` |
| 自动拆箱 | 包装类型 → 基本类型 | `Integer.intValue()` |

**应用场景：**
- 集合存储基本类型（`List<Integer>`）
- 方法参数传递（泛型方法）
- 算术运算（包装类型参与运算）

## 13. 什么是Java中的继承机制

### 📌 回答重点

**继承**是Java面向对象的核心特性，允许子类复用父类的字段和方法，用 `extends` 关键字建立继承关系。

**核心特点：**
- Java只支持**单继承**，类最多只能有一个直接父类
- 但可以通过继承链形成多层结构
- 子类能继承父类的 `public` 和 `protected` 成员
- `private` 成员虽然存在于子类对象内存中，但无法直接访问

---

### 💡 面试官最爱问的3个细节

#### 1) 继承的基本示例

```java
// 父类
public class Animal {
    protected String name;
    
    public void eat() {
        System.out.println(name + " is eating");
    }
}

// 子类
public class Dog extends Animal {
    public Dog(String name) {
        this.name = name;  // 继承了父类的name字段
    }
    
    public void bark() {
        System.out.println(name + " is barking");
    }
}

// 使用
Dog dog = new Dog("旺财");
dog.eat();   // 调用继承的方法
dog.bark();  // 调用自己的方法
```

---

#### 2) 构造方法的执行顺序

**关键规则：**
- 子类构造方法执行前，必须先调用父类构造方法
- 如果父类有无参构造，编译器自动加上 `super()`
- 如果父类只有带参构造，子类必须显式调用 `super(参数)`

```java
// 情况1：父类有无参构造
public class Animal {
    public Animal() {
        System.out.println("Animal构造方法");
    }
}

public class Dog extends Animal {
    public Dog() {
        super();  // 编译器自动添加，可省略
        System.out.println("Dog构造方法");
    }
}

// 执行顺序：Animal构造方法 → Dog构造方法

// 情况2：父类只有带参构造
public class Animal {
    private String name;
    
    public Animal(String name) {  // 没有无参构造
        this.name = name;
    }
}

public class Dog extends Animal {
    public Dog(String name) {
        super(name);  // 必须显式调用，否则编译报错！
    }
}
```

---

#### 3) 访问权限与继承

| 修饰符 | 本类 | 同包 | 子类 | 其他 |
|--------|------|------|------|------|
| public | ✅ | ✅ | ✅ | ✅ |
| protected | ✅ | ✅ | ✅ | ❌ |
| default | ✅ | ✅ | ❌ | ❌ |
| private | ✅ | ❌ | ❌ | ❌ |

```java
public class Father {
    public int publicField = 1;
    protected int protectedField = 2;
    int defaultField = 3;          // 包访问权限
    private int privateField = 4;  // 私有字段
}

public class Son extends Father {
    public void access() {
        System.out.println(publicField);      // ✅ 可以访问
        System.out.println(protectedField);   // ✅ 可以访问
        // System.out.println(defaultField);  // ❌ 不同包无法访问
        // System.out.println(privateField);  // ❌ private无法访问
        
        // private字段只能通过父类的public方法间接访问
    }
}
```

---

### 📝 总结

| 特性 | 说明 |
|------|------|
| 关键字 | `extends` |
| 继承类型 | 单继承（一个类只能有一个直接父类） |
| 可继承成员 | `public`、`protected`成员 |
| 不可继承 | `private`成员、构造方法 |
| 构造顺序 | 父类构造 → 子类构造 |

### 🔍 拓展知识：继承的利弊

**优点：**

## 15. Java中静态方法和实例方法的区别是什么

### 📌 回答重点

**核心区别就一句话：** 静态方法属于类，不需要对象就能调用；实例方法属于对象，必须先 `new` 出来才能用。

**静态方法（static）：**
- 用 `static` 修饰
- 通过类名直接调用，如 `Math.sqrt(4)`
- 类加载时就存在，不依赖任何对象实例
- 不能用 `this`，也不能直接访问实例变量和实例方法

**实例方法：**
- 不带 `static`
- 必须通过对象调用，如 `user.getName()`
- 能访问当前对象的实例变量
- 也能调用静态方法和静态变量（权限比静态方法大）

---

### 💡 面试官最爱问的3个细节

#### 1) 静态方法示例：工具类

```java
// 静态方法：工具类，不需要状态
public class StringUtils {
    public static boolean isEmpty(String str) {
        return str == null || str.length() == 0;
    }
    
    public static String reverse(String str) {
        if (str == null) return null;
        return new StringBuilder(str).reverse().toString();
    }
}

// 调用：StringUtils.isEmpty(name)
```

| 特性 | 静态方法 | 实例方法 |
|------|----------|----------|

| 内存分配 | 方法区 | 堆+方法区 |

### 🔍 拓展知识：访问权限

#### 静态方法无法访问实例成员

```java
public class Example {
    private int instanceVar = 10;
    private static int staticVar = 20;
    
    // ❌ 错误：静态方法不能访问实例变量
    public static void staticMethod() {
        // System.out.println(instanceVar);  // 编译错误
        System.out.println(staticVar);       // ✅ 正确
    }
    
    // ✅ 正确：实例方法可以访问静态变量
    public void instanceMethod() {
        System.out.println(instanceVar);    // ✅ 正确
        System.out.println(staticVar);       // ✅ 正确
    }
}
```

#### 实例方法可以调用静态方法

```java
public class Example {
    public static void staticMethod() {
        System.out.println("静态方法");
    }
    
    public void instanceMethod() {
        staticMethod();  // ✅ 实例方法可以调用静态方法
    }
}
```

### ⚠️ 常见陷阱

#### 陷阱1：静态方法中使用this

```java
public class Example {
    private String name;
    
    // ❌ 错误：静态方法中不能使用this
    public static void staticMethod() {
        // this.name = "test";  // 编译错误
    }
    
    // ✅ 正确：实例方法中使用this
    public void instanceMethod() {
        this.name = "test";  // 正确
    }
}
```

#### 陷阱2：静态方法访问实例方法

```java
public class Example {
    public void instanceMethod() {
        System.out.println("实例方法");
    }
    
    // ❌ 错误：静态方法不能直接调用实例方法
    public static void staticMethod() {
        // instanceMethod();  // 编译错误
    }
    
    // ✅ 正确：先创建对象再调用
    public static void staticMethodCorrect() {
        Example example = new Example();
        example.instanceMethod();  // 正确
    }
}
```

#### 陷阱3：静态方法无法重写

```java
public class Parent {
    public static void staticMethod() {
        System.out.println("Parent");
    }
}

public class Child extends Parent {
    // ❌ 这不是重写，而是隐藏
    public static void staticMethod() {
        System.out.println("Child");
    }
}

// 调用时看引用类型
Parent parent = new Child();
parent.staticMethod();  // 输出：Parent（静态方法不参与多态）
```
