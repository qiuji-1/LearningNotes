## 一、Markdown 简介

### 1.1 什么是 Markdown？

**Markdown** 是一种轻量级标记语言，由 **John Gruber** 于 **2004年** 创建。它允许人们使用易读易写的纯文本格式编写文档，然后转换成结构化的 HTML 或其他格式。

### 1.2 核心特点

| 特点 | 说明 |
|------|------|
| **纯文本格式** | 就像写普通文本，但可以用简单符号添加格式 |
| **易读易写** | 语法简单直观，即使不看渲染效果也能理解内容 |
| **跨平台** | 任何文本编辑器都能打开，不依赖特定软件 |
| **版本控制友好** | 可以用 Git 追踪修改历史，比 Word 文件更适合代码管理 |
| **专注内容** | 让你专注于写作内容，而不是排版格式 |

### 1.3 文件扩展名

Markdown 文件的扩展名通常是 `.md` 或 `.markdown`。

## 二、应用场景

### 2.1 技术领域

- **项目文档** - README.md、API 文档、开发指南
- **技术博客** - GitHub Pages、Hexo、Hugo 等静态博客
- **代码注释** - 代码仓库中的说明文档
- **开源项目** - 贡献指南、变更日志 (CHANGELOG)

### 2.2 其他用途

- **个人笔记** - Obsidian、Notion、Typora 等笔记软件
- **在线协作** - GitHub、GitLab 的 Wiki 和 Issue
- **书籍写作** - GitBook、技术书籍
- **演示文稿** - Marp、reveal.js 等工具

## 三、Markdown vs Word vs .txt

| 对比项 | Markdown | Word | .txt |
|--------|----------|------|------|
| **文件大小** | 几 KB | 几百 KB ~ MB | 几 KB（最小） |
| **版本控制** | ✅ 完美支持（Git 友好） | ❌ 二进制文件，难以追踪修改 | ✅ 完美支持（Git 友好） |
| **跨平台** | ✅ 任何系统都能打开 | ⚠️ 需要安装 Office | ✅ 任何系统都能打开 |
| **技术文档** | ✅ 程序员标准选择 | ❌ 不适合代码嵌入 | ⚠️ 无格式，不适合复杂文档 |
| **图片处理** | ⚠️ 需要单独管理图片文件 | ✅ 图片直接嵌入文档 | ❌ 不支持图片 |
| **格式复杂度** | ⚠️ 适合简单格式 | ✅ 支持复杂排版（表格、分栏等） | ❌ 无格式支持 |
| **数学公式** | ✅ 支持 LaTeX 语法 | ⚠️ 公式编辑器较复杂 | ❌ 不支持公式 |
| **学习成本** | ✅ 低，15分钟上手 | ⚠️ 需要熟悉工具栏 | ✅ 零学习成本 |

### 适用场景建议

- **用 Markdown**：技术文档、代码项目、个人笔记、博客文章、版本控制的文档
- **用 Word**：正式论文、合同文件、复杂排版、需要打印的文档、协作编辑
- **用 .txt**：简单日志、临时记录、配置文件、无格式需求的纯文本

## 四、为什么程序员喜欢 Markdown？

1. **Git 友好** - 纯文本格式，可以清晰地看到每次修改的差异（diff）
2. **代码嵌入** - 可以直接写代码块，支持语法高亮
3. **专注写作** - 不用频繁切换鼠标点击工具栏，双手不离键盘
4. **可编程** - 可以用脚本批量处理、转换、生成
5. **社区标准** - GitHub、Stack Overflow、技术博客都用 Markdown

## 五、学习建议

### 5.1 学习路径

1. **第一步**：掌握基础语法（标题、列表、链接、图片、代码块）
2. **第二步**：学习进阶语法（表格、引用、分割线、转义字符）
3. **第三步**：了解扩展语法（任务列表、流程图、数学公式）
4. **第四步**：实际应用（写 README、写技术博客、记笔记）

### 5.2 推荐工具

- **编辑器**：
  - VS Code（免费、强大）
  - Typora（所见即所得，收费）
