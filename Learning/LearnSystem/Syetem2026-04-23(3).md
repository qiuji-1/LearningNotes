## 十五、进程间通信 (IPC)

### 15.1 为什么需要IPC？

进程之间**内存隔离**，不能直接互相访问数据，必须通过操作系统提供的"官方渠道"交换信息。

> 类比 🏭：两个工厂之间有围墙，A工厂的人不能直接走进B工厂拿东西，必须通过官方渠道。

### 15.2 七种通信方式

#### 方式1：管道 (Pipe)
```
┌──────────┐         ┌──────────┐
│  进程A   │  ──▶    │  进程B   │
│  (写端)   │  管道    │  (读端)   │
└──────────┘         └──────────┘
```
- 单向传输，像一根水管
- 内存中的临时通道，进程结束就消失
- 只能用于**有亲缘关系**的进程（父子进程）

#### 方式2：命名管道 (Named Pipe / FIFO)
| | 普通管道 | 命名管道(FIFO) |
|--|---------|---------------|
| 有名字？ | ❌ 无名 | ✅ 有文件路径 |
| 谁能用？ | 父子进程 | **任意无关进程** |
| 生命周期 | 随进程消失 | 删除文件才消失 |

#### 方式3：消息队列 (Message Queue)
```
        ┌──────────────┐
  写入▶  │              │  ▶读取
进程A ──┤  消息队列      ├── 进程B
        │  (内核维护)    │
进程C ──┤              ├── 进程D
  写入▶  └──────────────┘  ▶读取
```
- 内核维护的一个"消息链表"
- 多个进程可以同时往里面写、从里面读
- 每条消息有类型，可按类型选择性接收
- 发送方不用等接收方确认（异步）
> 类比 📮：邮局信箱——信扔进去就走，收件人自己来取。

#### 方式4：共享内存 (Shared Memory) — ⚡最快！
```
┌──────────────────────────────────┐
│       共享内存区域                │
│       (内核分配的一块内存)        │
│   ┌──────┐           ┌──────┐    │
│   │进程A │  直接读写  │进程B │    │
│   └──────┘ ◀──────▶ └──────┘    │
└──────────────────────────────────┘
```
- ⚡ **速度最快**，数据不需要拷贝
- OS在内存中划一块区域，两个进程都能直接访问
- ⚠️ 需要自己加锁/同步机制（不然同时改会乱）
> 类比 🏭：两家人共用一个仓库，得约定好规矩一次只能进一家。

#### 方式5：信号量 (Semaphore) — 协调信号灯
- 不传数据，只做协调（计数器 + 等待队列）
- 值=1 绿灯可通过；值=0 红灯排队等
- 通常和**共享内存配合使用**
> 类比 🚦：停车场空位计数器——有位放行，没位排队，走一辆进一辆。

#### 方式6：信号 (Signal) — 异常通知
- 传送的信息极小（只有一个整数编号）
- 用于通知/控制，不用于传数据
- 异步的（收到信号时当前工作会被打断）

常见信号：

| 信号 | 含义 |
|------|------|
| SIGKILL (9) | 杀掉进程，无法忽略 |
| SIGSTOP (19) | 暂停进程 |
| SIGSEGV (11) | 段错误（非法内存访问） |
| SIGINT (2) | Ctrl+C 中断 |

> 类比 🔔：工厂警报器——拉响某个警报表示发生了某件事。

#### 方式7：Socket — 唯一能跨网络的方式！
- ✅ 本地用（Unix Domain Socket）
- ✅ **跨网络跨机器用**（TCP/IP Socket）← 最大优势
- 其他6种都限制在同一台机器上

> 类比 📞：其他方式是工厂内部传话筒；Socket是电话——能打到别的城市。

### 15.3 总结对比

| 方式 | 传输方向 | 能否跨网络 | 速度 | 适用场景 |
|------|---------|-----------|------|---------|
| 管道 | 单向 | ❌ 仅本机 | 快 | 父子进程简单通信 |
| 命名管道 | 单向 | ❌ 仅本机 | 快 | 无关进程简单通信 |
| 消息队列 | 双向 | ❌ 仅本机 | 中 | 多进程异步通信 |
| 共享内存 | 双向 | ❌ 仅本机 | ⚡最快 | 大量数据传输 |
| 信号量 | N/A | ❌ 仅本机 | N/A | 进程同步协调 |
| 信号 | N/A | ❌ 仅本机 | 最快 | 通知/控制 |
| Socket | 双向 | ✅ **跨网络** | 较慢 | 分布式/网络通信 |

### 15.4 选择指南

```
大量数据 + 追求速度？     → 共享内存（配信号量同步）
多进程异步发消息？        → 消息队列
父子进程简单传数据？      → 管道
不同机器上的进程通信？     → Socket ← 没得选
只需要通知一下？          → 信号
```

## 十六、管道的"亲缘关系"在代码中如何体现

### 16.1 父子进程靠 `fork()` 创建

```c
#include <unistd.h>
#include <stdio.h>

int main() {
    printf("我是父进程, PID=%d\n", getpid());
    
    pid_t pid = fork();   // ← ★ 关键！创建子进程
    
    if (pid == 0) {
        // 子进程（fork返回0）
        printf("我是子进程, 父PID=%d\n", getppid());
    } else {
        // 父进程（fork返回子进程的PID）
        printf("我(父进程)创建了子进程, PID=%d\n", pid);
    }
}
```
**fork() 的本质：把父进程完整复制一份变成子进程**

| 属性 | 父进程 | 子进程 |
|------|--------|--------|
| PID | 原来的(如1001) | 新分配的(如1002) |
| `fork()` 返回值 | 子进程PID | **0** |
| 内存数据 | [ABC] | [ABC]（复制） |

### 16.2 为什么匿名管道只能用于父子进程？

管道的文件描述符(fd)是通过 **fork继承** 得到的：

```c
int pipefd[2];
pipe(pipefd);            // ① 先创建管道

pid_t pid = fork();      // ② 再fork子进程

if (pid == 0) {
    // 子进程：继承了相同的pipefd，从读端读
    close(pipefd[1]);
    read(pipefd[0], buf, sizeof(buf));
} else {
    // 父进程：往写端写
    close(pipefd[0]);
    write(pipefd[1], "hello", 5);
}
```

**只有通过 fork 继承了相同 fd 的两个进程，才能知道同一个管道在哪！**
无关进程没有这个 fd → 找不到管道 → 无法使用。

### 16.3 匿名管道 vs 命名管道 的核心区别

| | 匿名管道 | 命名管道(FIFO) |
|--|---------|---------------|
| 怎么找到管道？ | 靠 **fork继承** 的fd | 靠 **文件系统路径** `/tmp/my_pipe` |
| 谁能用？ | 只有继承fd的父子进程 | **任何人都能按路径open** |

```
匿名管道：  "我有你家的钥匙"（继承来的）→ 能进门
            ↑ 只有家人(fork出的)才有钥匙

命名管道：  "我知道你家地址"（文件路径）→ 谁都能来敲门
            ↑ 地址是公开的，谁都可以来
```

### 16.4 Java 中类似的概念

```java
// ProcessBuilder 创建子进程（类似fork的效果）
Process p = new ProcessBuilder("ls", "-la").start();

// getInputStream() 相当于管道的读端
InputStream input = p.getInputStream();
// 读取子进程的输出
```

### 16.5 概念对比总结

| 概念 | 代码体现 | 类比 |
|------|---------|------|
| 类的继承 | `extends` | 儿子继承父亲的基因 |
| 进程的亲缘关系 | `fork()` | 老板招聘员工（复制自己） |
| 匿名管道 | `pipe()` + `fork()` | 内部传话筒（自家人用） |
| 命名管道 | `mkfifo()` + `open(path)` | 公共信箱（谁都能投递） |
