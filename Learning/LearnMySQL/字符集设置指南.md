## 📚 字符集基础概念

### 什么是字符集？

字符集（Character Set）是数据库用来存储和显示文字的编码规则。不同的字符集支持不同的字符范围和编码方式。

### 常见的MySQL字符集对比

| 字符集 | 说明 | 支持字符 | 缺点 | 推荐度 |
|--------|------|---------|------|--------|
| **utf8** | MySQL的"假"UTF-8 | 最多3字节 | ❌ 无法存储emoji、部分汉字 | ⭐⭐ |
| **utf8mb4** | 真正的UTF-8 | 最多4字节 | ✅ 支持所有Unicode字符（包括emoji） | ⭐⭐⭐⭐⭐ |
| **gbk** | 中文字符集 | 仅支持中文 | ❌ 不支持其他语言 | ⭐⭐ |
| **latin1** | 西欧字符集 | 仅支持西欧语言 | ❌ 不支持中文 | ⭐ |

### 为什么推荐utf8mb4？

1. **MySQL的`utf8`不是真正的UTF-8**，只能存储3字节字符
2. **`utf8mb4`才是真正的UTF-8**，可以存储4字节字符
3. **可以正确存储emoji、繁体字、特殊符号等**
4. **国际化标准**，支持所有Unicode字符

**示例对比**：
```sql
-- 使用 utf8（无法存储emoji）
CREATE TABLE test_utf8 (
    content VARCHAR(100) CHARACTER SET utf8
) ENGINE=InnoDB;
INSERT INTO test_utf8 VALUES ('你好😊');  -- ❌ 错误：数据截断

-- 使用 utf8mb4（可以存储emoji）
CREATE TABLE test_utf8mb4 (
    content VARCHAR(100) CHARACTER SET utf8mb4
) ENGINE=InnoDB;
INSERT INTO test_utf8mb4 VALUES ('你好😊');  -- ✅ 成功
```

## 🔧 如何设置utf8mb4

### 1. 创建数据库时指定字符集

```sql
-- 推荐写法
CREATE DATABASE IF NOT EXISTS test_db 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;

-- 验证
SHOW CREATE DATABASE test_db;
```

### 2. 修改现有数据库字符集

```sql
ALTER DATABASE test_db 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
```

### 3. 创建表时指定字符集

```sql
-- 方式1：在表级别指定
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50),
    email VARCHAR(100),
    age INT
) ENGINE=InnoDB 
DEFAULT CHARSET=utf8mb4 
COLLATE=utf8mb4_unicode_ci;

-- 方式2：在字段级别指定
CREATE TABLE products (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci,
    description TEXT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci
) ENGINE=InnoDB;
```

### 4. 修改现有表字符集

```sql
-- 修改表的默认字符集
ALTER TABLE users 
CONVERT TO CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;

-- 修改特定字段的字符集
ALTER TABLE users 
MODIFY COLUMN name VARCHAR(50) CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## 📊 COLLATE（排序规则）详解

### 什么是COLLATE？

COLLATE（排序规则）定义了字符的比较和排序方式。它决定了：
- 字符如何排序（ORDER BY）
- 字符如何比较（WHERE、GROUP BY）
- 是否区分大小写

### 常见的utf8mb4排序规则

| 排序规则 | 说明 | 区分大小写 | 性能 | 准确性 | 推荐场景 |
|---------|------|-----------|------|--------|---------|
| `utf8mb4_unicode_ci` | Unicode排序 | ❌ 否 | 中 | ✅ 高 | 通用场景，推荐 |
| `utf8mb4_general_ci` | 通用排序 | ❌ 否 | ✅ 高 | 中 | 追求性能 |
| `utf8mb4_bin` | 二进制排序 | ✅ 是 | ✅ 高 | ✅ 最高 | 精确匹配 |

**后缀含义**：
- `_ci`：Case Insensitive（不区分大小写）
- `_cs`：Case Sensitive（区分大小写）
- `_bin`：Binary（二进制，严格区分）

### 排序规则使用示例

**测试数据准备**：
```sql
-- 创建测试表并插入数据（包含不同大小写的英文名）
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

INSERT INTO users (name) VALUES 
('Alice'),
('alice'),
('ALICE'),
('Bob'),
('bob');
```

**不区分大小写排序（_ci）**：
```sql
-- 会匹配 Alice、alice、ALICE 三个记录
SELECT * FROM users 
WHERE name = 'alice'
ORDER BY name;
```
**结果**：
```
+----+-------+
| id | name  |
+----+-------+
|  1 | Alice |
|  2 | alice |
|  3 | ALICE |
+----+-------+
```
✅ 因为`_ci`不区分大小写，所以 `alice` = `Alice` = `ALICE`

**区分大小写排序（_bin）**：
```sql
-- 只匹配小写的 alice
SELECT * FROM users 
WHERE name COLLATE utf8mb4_bin = 'alice'
ORDER BY name COLLATE utf8mb4_bin;
```
**结果**：
```
+----+-------+
| id | name  |
+----+-------+
|  2 | alice |
+----+-------+
```
✅ 因为`_bin`是二进制匹配，严格区分大小写，只有小写的`alice`匹配

## 🔍 检查字符集设置

### 1. 检查数据库字符集

```sql
-- 查看所有数据库的字符集
SELECT 
    SCHEMA_NAME AS '数据库名',
    DEFAULT_CHARACTER_SET_NAME AS '字符集',
    DEFAULT_COLLATION_NAME AS '排序规则'
FROM information_schema.SCHEMATA
WHERE SCHEMA_NAME = 'test_db';

-- 或者
SHOW CREATE DATABASE test_db;
```

**输出示例**：
```
+---------+-----------+-------------------+
| 数据库名 | 字符集    | 排序规则           |
+---------+-----------+-------------------+
| test_db | utf8mb4   | utf8mb4_unicode_ci |
+---------+-----------+-------------------+
```

### 2. 检查表字符集

```sql
-- 查看特定数据库中所有表的字符集
SELECT 
    TABLE_NAME AS '表名',
    TABLE_COLLATION AS '排序规则'
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'test_db';

-- 或者
SHOW TABLE STATUS FROM test_db;

-- 查看特定表
SHOW CREATE TABLE users;
```

### 3. 检查字段字符集

```sql
-- 查看表中所有字段的字符集
SHOW FULL COLUMNS FROM users;

-- 或者
SELECT 
    COLUMN_NAME AS '字段名',
    CHARACTER_SET_NAME AS '字符集',
    COLLATION_NAME AS '排序规则'
FROM information_schema.COLUMNS
WHERE TABLE_SCHEMA = 'test_db' 
AND TABLE_NAME = 'users';
```

### 4. 检查连接字符集

```sql
-- 查看当前连接的字符集设置
SHOW VARIABLES LIKE 'character_set%';
SHOW VARIABLES LIKE 'collation%';
```

**关键变量说明**：
| 变量名 | 说明 |
|--------|------|
| `character_set_client` | 客户端发送查询时使用的字符集 |
| `character_set_connection` | 连接使用的字符集 |
| `character_set_database` | 当前数据库的字符集 |
| `character_set_results` | 返回结果时使用的字符集 |
| `character_set_server` | MySQL服务器默认字符集 |
| `character_set_system` | 系统元数据使用的字符集（总是utf8） |

**理想状态**：所有都是 `utf8mb4`

## 💻 连接时指定字符集

### 命令行连接

```bash
# 方式1：在连接时指定字符集
mysql -u root -p --default-character-set=utf8mb4

# 方式2：在连接后设置字符集
mysql -u root -p
mysql> SET NAMES utf8mb4;
```

### 配置文件设置（my.ini 或 my.cnf）

```ini
[client]
default-character-set=utf8mb4

[mysql]
default-character-set=utf8mb4

[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci

[mysql_safe]
default-character-set=utf8mb4
```

**修改配置文件后需要重启MySQL服务**：
```bash
# Windows
net stop mysql
net start mysql

# Linux
sudo systemctl restart mysql
```

### JDBC连接字符串

```java
// Java JDBC连接
String url = "jdbc:mysql://localhost:3306/test_db" +
    "?useUnicode=true" +
    "&characterEncoding=utf8mb4" +
    "&useSSL=false";
```

## ⚠️ 常见问题

### Q1: 为什么MySQL的utf8无法存储emoji？

**A**: MySQL的`utf8`字符集实际上只支持3字节的UTF-8编码，而emoji需要4字节编码。应该使用`utf8mb4`。

### Q2: utf8mb4比utf8慢吗？

**A**: 理论上`utf8mb4`略慢（因为支持更多字符），但实际差异非常小，几乎可以忽略。推荐使用`utf8mb4`。

### Q3: 如何选择排序规则？

**A**:
- 通用场景：`utf8mb4_unicode_ci`（推荐）
- 追求性能：`utf8mb4_general_ci`
- 需要精确匹配：`utf8mb4_bin`（区分大小写）

### Q4: 修改字符集会影响现有数据吗？

**A**: 
- 使用`ALTER DATABASE`不会影响现有表
- 使用`ALTER TABLE ... CONVERT TO`会转换表中所有数据的字符集，可能丢失数据，**务必先备份**

### Q5: 如何修复乱码问题？

**A**: 
```sql
-- 1. 检查当前字符集
SHOW VARIABLES LIKE 'character_set%';

-- 2. 设置连接字符集
SET NAMES utf8mb4;

-- 3. 重新查询
SELECT * FROM users;
```

### Q6: 为什么查询结果显示问号？

**A**: 可能是字符集不匹配。确保：
- 数据库字符集是`utf8mb4`
- 表字符集是`utf8mb4`
- 连接字符集是`utf8mb4`
- 客户端显示编码是`UTF-8`
