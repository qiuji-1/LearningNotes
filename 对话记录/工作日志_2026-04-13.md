### 6.1.1 用户认证模块

用户认证模块是系统的入口功能，负责用户身份的验证与管理，包括登录和注册两个核心功能。

**登录功能**

登录页面是用户进入系统的第一个界面。如图6-1所示，登录页面主要包含以下界面元素：

- 顶部Logo区域：显示系统名称"个人健康管理系统"，起到品牌识别作用
- 表单输入区域：包含账号输入框和密码输入框，均设有输入提示信息
- 功能按钮区域：包含"登录"按钮和"去注册"链接

【图6-1 用户登录页面】

**交互逻辑说明：**

1. 用户登录：用户在账号和密码输入框中填写信息，点击"登录"按钮后，系统对输入内容进行非空校验，校验通过后将密码进行MD5加密传输至后端。后端验证账号密码正确性，若验证成功，根据用户角色（普通用户或管理员）跳转至对应首页，并保存JWT令牌用于后续请求的身份认证；若验证失败，页面弹出错误提示。

2. 跳转注册：点击"去注册"链接，页面跳转至注册页面。

**注册功能**

注册页面为新用户提供账号创建功能。如图6-2所示，注册页面主要包含以下界面元素：

- 表单区域：包含账号、密码、确认密码、手机号、邮箱等输入项，每项均设有格式校验提示
- 头像上传区域：支持用户选择本地图片作为头像
- 提交按钮：点击完成注册

【图6-2 用户注册页面】

**交互逻辑说明：**

1. 表单校验：用户填写注册信息时，系统实时校验各字段格式。账号检验唯一性，密码需两次输入一致，手机号和邮箱需符合正则表达式规则。

2. 提交注册：用户点击"注册"按钮后，系统进行完整表单校验，校验通过后将数据提交至后端。后端再次验证账号唯一性后创建用户记录，返回注册成功提示，页面自动跳转至登录页面。

# 第5章 系统实现

## 5.1 用户端功能实现

### 5.1.1 用户认证模块

用户认证模块是系统的入口功能，负责用户身份的验证与管理，包括登录和注册两个核心功能。

**登录功能**

登录页面是用户进入系统的第一个界面。如图5-1所示，登录页面主要包含以下界面元素：

- 顶部Logo区域：显示系统名称"个人健康管理系统"，起到品牌识别作用
- 表单输入区域：包含账号输入框和密码输入框，均设有输入提示信息
- 功能按钮区域：包含"登录"按钮和"去注册"链接

【图5-1 用户登录页面】

**交互逻辑说明：**

1. 用户登录：用户在账号和密码输入框中填写信息，点击"登录"按钮后，系统对输入内容进行非空校验，校验通过后将密码进行MD5加密传输至后端。后端验证账号密码正确性，若验证成功，根据用户角色（普通用户或管理员）跳转至对应首页，并保存JWT令牌用于后续请求的身份认证；若验证失败，页面弹出错误提示。

2. 跳转注册：点击"去注册"链接，页面跳转至注册页面。

**注册功能**

注册页面为新用户提供账号创建功能。如图5-2所示，注册页面主要包含以下界面元素：

- 表单区域：包含账号、密码、确认密码、手机号、邮箱等输入项，每项均设有格式校验提示
- 头像上传区域：支持用户选择本地图片作为头像
- 提交按钮：点击完成注册

【图5-2 用户注册页面】

**交互逻辑说明：**

1. 表单校验：用户填写注册信息时，系统实时校验各字段格式。账号检验唯一性，密码需两次输入一致，手机号和邮箱需符合正则表达式规则。

2. 提交注册：用户点击"注册"按钮后，系统进行完整表单校验，校验通过后将数据提交至后端。后端再次验证账号唯一性后创建用户记录，返回注册成功提示，页面自动跳转至登录页面。

### 5.1.2 用户首页模块

用户首页是普通用户登录后的主界面，展示系统核心功能入口和推荐内容。

如图5-3所示，用户首页主要包含以下区域：

- 顶部导航栏：包含系统Logo、搜索框、用户头像及下拉菜单（个人中心、退出登录）
- 轮播图区域：展示健康主题的轮播图片，起到宣传和引导作用
- 热门资讯推荐区：基于协同过滤算法为用户推荐的健康资讯卡片
- 快捷入口区：包含健康数据、食谱、饮食记录等功能的快捷入口按钮
- 右侧边栏：展示最新资讯列表和热门食谱推荐

【图5-3 用户首页】

**交互逻辑说明：**

1. 浏览资讯：用户点击热门资讯推荐区的卡片，页面跳转至资讯详情页，展示完整文章内容。

2. 快捷入口：点击"健康数据"快捷入口，跳转至健康数据管理页面；点击"食谱"快捷入口，跳转至食谱列表页面。

3. 用户菜单：点击右上角用户头像，展开下拉菜单，选择"个人中心"跳转至个人信息管理页面，选择"退出登录"清除登录状态并返回登录页面。

### 5.1.3 个人健康数据模块

个人健康数据模块是系统的核心功能模块之一，支持用户配置健康指标模组、录入健康数据和查看数据可视化图表。

**健康模组配置**

健康模组配置页面允许用户自定义关注的健康指标项。如图5-4所示，页面主要包含以下区域：

- 可选模组列表：展示系统预设的健康模组（如身高、体重、血压、心率等），每个模组显示名称、单位、正常范围信息
- 已选模组区域：展示用户当前已关注配置的健康模组
- 新增按钮：支持用户创建自定义健康模组

【图5-4 健康模组配置页面】

**交互逻辑说明：**

1. 选择模组：用户在可选模组列表中点击某个模组卡片，该模组添加至已选区域，用户可在健康记录页面使用该模组录入数据。

2. 取消选择：用户在已选区域点击模组卡片上的"×"按钮，该模组从已选区域移除，不再显示在健康记录页面。

3. 新增自定义模组：点击"新增"按钮，页面右侧弹出抽屉弹窗，用户填写模组名称、单位、正常范围上下限，点击"保存"后，自定义模组添加至已选区域。

**健康数据录入**

健康数据录入页面用于记录用户的健康指标数值。如图5-5所示，页面主要包含以下区域：

- 模组选择下拉框：用户选择要录入数据的健康模组
- 数值输入框：用户输入本次测量的数值
- 正常范围提示：显示当前模组的正常范围，帮助用户判断数据是否异常
- 提交按钮：保存健康记录
- 历史记录列表：展示用户的历史健康记录，包含时间、数值、状态（正常/偏高/偏低）

【图5-5 健康数据录入页面】

**交互逻辑说明：**

1. 录入数据：用户选择模组后，页面显示该模组的正常范围提示。用户输入数值，点击"提交"按钮，系统根据预设阈值自动判断数据状态（正常、偏高、偏低），并在历史记录列表中以不同颜色标识。

2. 查看详情：点击历史记录列表中的某条记录，展开显示详细信息，包括录入时间、数值、状态判定结果。

**健康数据可视化**

健康数据可视化页面使用ECharts图表库展示健康数据变化趋势。如图5-6所示，页面主要包含以下区域：

- 模组选择区域：用户选择要查看的健康模组
- 时间范围选择器：提供"近一周"、"近一月"、"近三月"、"近一年"四个时间范围选项
- 图表展示区：以折线图或柱状图形式展示健康数据随时间的变化趋势
- 数据统计区：展示选定时间范围内的平均值、最大值、最小值统计

【图5-6 健康数据可视化页面】

**交互逻辑说明：**

1. 切换模组：用户选择不同的健康模组，图表自动更新为对应模组的数据展示。

2. 切换时间范围：用户点击时间范围选择器中的某个选项，图表重新加载对应时间段的数据，折线图横轴时间刻度相应调整。

### 5.1.4 健康资讯浏览模块

健康资讯浏览模块为用户提供健康知识文章的浏览和阅读功能，并基于协同过滤算法推荐个性化内容。

**资讯列表页**

如图5-7所示，资讯列表页主要包含以下区域：

- 分类筛选栏：展示资讯分类标签（如饮食健康、运动健身、心理健康等），用户可按分类筛选
- 资讯卡片列表：以卡片形式展示资讯缩略图、标题、摘要、发布时间
- 分页器：当资讯数量超过每页显示上限时，提供分页导航

【图5-7 资讯列表页面】

**交互逻辑说明：**

1. 分类筛选：用户点击分类筛选栏中的某个分类标签，列表刷新为该分类下的资讯内容。

2. 浏览详情：用户点击资讯卡片，页面跳转至资讯详情页，展示完整文章内容。

3. 分页浏览：用户点击分页器中的页码或"下一页"按钮，列表加载对应页的资讯内容。

**资讯详情页**

如图5-8所示，资讯详情页主要包含以下区域：

- 文章正文区：展示资讯标题、发布时间、作者、正文内容
- 收藏按钮：位于标题右侧，用户可收藏感兴趣的资讯
- 推荐阅读区：基于协同过滤算法推荐的相关资讯列表
- 评论区：位于文章底部，展示用户评论列表和评论输入框

【图5-8 资讯详情页面】

**交互逻辑说明：**

1. 收藏资讯：用户点击收藏按钮，按钮图标变为已收藏状态，页面顶部弹出"收藏成功"提示。再次点击取消收藏。

2. 发表评论：用户在评论输入框中输入评论内容，点击"发表"按钮，系统使用Trie字典树算法检测敏感词，若包含敏感词则提示修改，若通过检测则评论显示在评论列表中。

3. 点赞评论：用户点击某条评论下方的点赞按钮，点赞数加1，按钮变为已点赞状态。

```java
// Trie字典树敏感词检测
public boolean containsSensitiveWord(String text) {
    for (int i = 0; i < text.length(); i++) {
        int matchLength = checkFromPosition(text, i);
        if (matchLength > 0) {
            return true; // 检测到敏感词
        }
    }
    return false;
}

private int checkFromPosition(String text, int begin) {
    TrieNode node = rootNode;
    int matchLength = 0;
    for (int i = begin; i < text.length(); i++) {
        char c = text.charAt(i);
        node = node.getChild(c);
        if (node == null) break;
        matchLength++;
        if (node.isEnd()) return matchLength;
    }
    return 0;
}
```
